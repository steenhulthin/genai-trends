# Project Goals

Use this file to capture the product goals that should remain readable by both humans and agents.

## Purpose

- Describe what the project is meant to achieve.
- State the target users or audience.
- Explain why the project should exist.

## Success Criteria

- List the outcomes that define success.
- Prefer measurable or testable statements where possible.

## Non-Goals

- List things the project should explicitly not try to solve.

## Constraints

- Capture business, product, technical, or operational constraints that affect decisions.

## Open Questions

- Track unresolved product questions that may block implementation or require explicit decisions.
